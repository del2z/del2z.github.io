Vim入门指导

Vim是当前最流行的编辑器之一，从初版发布以来的近30年里，一直占据着主流编辑器的一席之地。尽管Vim的学习曲线很陡峭，但它独有的特性却始终吸引着一代代的程序员。在追求捷径和爆发式成功的当下环境中，Vim所代表的沉淀与厚重更显难能可贵。本篇就简单揭示Vim瑰丽世界的一角，以供品鉴。

常用模式

Vim常用的模式有三种，Normal、Insert、Visual、Command模式等。四种模式有不同的进入方法，如下表所示。

操作
Normal => Insert i 光标处插入 I 当前行第一个非空字符前插入 a 光标后插入 A 当前行行尾插入 o 在当前行之后开启新行插入 O 在当前行之前开启新行插入
Insert => Normal <Esc>或<C-c>
Normal => Visual v 进入Visual模式，从光标处开始选择 <S-v> 进入行Visual模式，从光标所处的行开始选择 <C-v> 进入列Visual模式，从光标所处的列开始选择
Visual => Normal <Esc>或<C-c>
Visual + Edit ;normal <operation><motion>
Normal => Command :<command>
Command => Normal <Esc>或<C-c>

操作指令

Vim在Normal模式下的命令通常是<operation><motion>形式，<operation>定义一个操作指令，<motion>定义一个动作指令。 操作指令、修改、复制、粘贴等，动作指令一般是上下左右、前进、后退等光标移动的指令。

匹配模式

文本的搜索和替换一般都会用到正则匹配，搜索和替换的基本命令如下。

  • /<pattern>，在当前文件中搜索模式<pattern>。
    • /\C<pattern>，搜索模式大小写敏感，如模式centos不能匹配CentOS、centOS、Centos等。
    • /\c<pattern>，搜索模式大小写不敏感，如模式CentOS可以匹配centos、centOS、Centos等。
  • :[pos]s/<pattern1>/<pattern2>/[opt]，在当前文件中搜索模式<pattern1>，将其替换为<pattern2>。
    • pos是搜索替换的范围，%代表文档所有行；5代表第5行；3,20表示文档第3行到第20行；.代表当前行；$表示文档最后一行。
    • opt是替换选项，c表示替换前确认；g表示全局，替换/统计每行所有匹配项；n表示匹配项计数。

常用的正则匹配符如下表。

Regex 匹配
^ 当前行行首
$ 当前行行尾
\s 空格、Tab等任意一个空字符
\d 0-9任意一个数字字符
\+ 一个或多个重复模式
* 零个或多个重复模式
. 任意一个字符
[abc] 范围内任意一个字符
